1.把文件添加到库，用命令git add。（没有任何显示，说明添加成功。）
  把文件提交到库，用命令git commit。
2，git status命令可以告诉我们文件的状态（修改文件的内容有没有提交）
   git diff命令可以告诉我们上次修改文件的内容。
3.命令git commit -m "append GPL"可以对文件提交备注。
  git log命令可以查看历史记录.
4.git reset --hard HEAD^可以回退上一个版本
  --hard会回退到上个版本的已提交状态，
  --soft会回退到上个版本的未提交状态，
  --mixed会回退到上个版本已添加但未提交的状态。
5.git reflog可以返回后面的版本

6.git commit只提交缓存区的修改
7. git checkout -- readme.txt可以让这个文件回到最近一次git commit或git add时的状
态。
8.删除文件rm test.txt有两个选择：
  1.确实要从版本库中删除该文件，那就用命令git rm删掉，并且git commit
  2.删错了，用git checkout恢复（从来没有被添加到版本库就被删除的文件，是无法恢复
的！）
9.git checkout -b dev可以创建dev分支，并切换。（需要在git仓库才可以建立）
  ==（git branch dev    git checkout dev）
  ==（git switch -c dev）
  git switch master直接切换到已有的master分支
10.git merge命令用于合并指定分支到当前分支
11. git branch -d dev命令删除dev分支
    git branch -D feature-vulcan强行删除
12.git merge --no-ff -m "merge with no-ff" dev命令将dev分支的所有修改合并到当前master分支，同时强制生成一条合并记录，完整保留分支合并的历史轨迹
13. git stash可以把当前工作现场“储藏”起来，等以后恢复现场后继续工作：
    git stash list可以查看工作现场在哪里
    恢复：1.git stash pop恢复内容的同时把stash内容也删了
          2.用git stash apply恢复，但是恢复后，stash内容并不删除，需要用git stash drop来删除；

14.git remote可以查看远程库的信息
   git remote -v显示更详细的信息
   git remote rm origin可以删除已有的GitHub远程库:
15. git push origin master把该分支上的所有本地提交推送到远程库
    git checkout -b dev origin/dev创建本地dev分支
    git push origin dev 把dev分支push到远程
    git pull可以将远程仓库的代码拉到本地仓库合并
    git rebaseebase操作可以把本地未push的分叉提交历史整理成直线
16.命令git tag <name>可以打一个新标签
   命令git tag可查看所有标签
   git tag v0.9 f52c633可以提交以前的标签（标签不是按时间顺序列出，而是按字母排>序的）
   git show <tagname>可以查看标签信息
   git tag -a <tagname> -m "blablabla...可以指定说明文字
17.命令git tag -d <tagname>可以删除一个本地标签
   命令git push origin <tagname>可以推送一个本地标签
   命令git push origin --tags可以推送全部未推送过的本地标签
   命令git push origin :refs/tags/<tagname>可以删除一个远程标签
   (删除远程标签，要先从本地删除然后，从远程删除)
18.git config --global alias.co checkout可以配置别名
   （前用户的Git配置文件放在用户主目录下的一个隐藏文件.gitconfig中）
